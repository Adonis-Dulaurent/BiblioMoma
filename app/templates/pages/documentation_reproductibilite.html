Documentation and Reproducibility

Presentation

This application offers a bibliographic journey by associating works and artists with resources from OpenAlex. It has been designed to be adaptable to other museums or institutions.

Prerequisites

Before using this application, make sure you have:

  • Downloaded the repository available on GitHub and followed the ReadMe.
  • A server with Python 3.9+.
  • A relational database of your institution's artworks and artists.

House by the Railroad, Edward Hopper (1925)

Data Preparation:

In your database, you must have:

  • A table Artist with the field: DisplayName
  • A table Artworks with the field: Title

Note that these fields are mandatory, but you can enrich your data with birth dates of artists. To process the data, we recommend using software like Orange Data Mining or the Python library Pandas.

Interface with OpenAlex

Queries with OpenAlex are made using its API. The application sends the name of the artwork or an artist using the request library. It then retrieves the results and displays them to the user.

User Management:

To enhance user experience, we offer an authentication system that allows users to save their bibliography, export it, and access visualizations of their selections.

For this, you need to add two tables to your database:

  • A table user with the fields: id, pseudo, password, email.
  • A table cart with the fields: id, user_id, date_added, bibliography, authors, type, open_access, publication_date, publisher

The first table stores users, while the second keeps their bibliographies with the necessary metadata for visualizations.

Customization and Deployment:

You can and should adapt the application:

  • Modify the HTML templates.
  • Adjust the CSS to match your graphic charter.
  • Change OpenAlex queries according to your needs.
  • Deploy the application locally or on a server.

Usage by Visitors:

Users can:

  • Search for artworks and artists.
  • Add bibliographic references to a cart.
  • Export their selections.
